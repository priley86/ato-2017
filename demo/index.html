Atomic Design

Building a maintainable codebase

"I'm not a great programmer; I'm just a good programmer with great habits." - Kent Beck

  • Develop your personal recipee for clean code:
    • LINTER + FORMATTER + TESTS = CONSISTENT CODE
  • Comment any potentially confusing code, or better yet, rewrite the code so that it isn't confusing.
  • Locate code quickly, Identify the code at a glance, keep the Flattest structure you can, and Try to be DRY
  • Reduce surface area for bugs:
    • Often several components need to reflect the same changing data. We recommend lifting the shared state up to their closest common ancestor.
  • Ensure recipees are vetted as soon as the pull request is opened

Javascript Standard Style


A zero configuration, community led style convention for Javascript.

  • Automatically fixes most lint issues with standard --fix
  • Integrates with most IDEs
  • Used by GitHub, NPM, and many others

Prettier Code Formatter


Prettier enforces a consistent code style by parsing all code and re-printing the parsed abstract syntax tree with its own rules.

  • One of the first formatters to actually solve the maximum line length problem
  • Uses standard ES6/ES7 conventions when formatting JS
  • Supports Javascript, JSX, Flow/Typescript, CSS/LESS/SASS, JSON

const regexp = /test   value/   // ✗ avoid

// No ternary operators when simpler alternatives exist.
let score = val ? val : 0     // ✗ avoid 
let score = val || 0          // ✓ ok 
        
  // The left operand of relational operators must not be negated.  
  if (!key in obj) {}       // ✗ avoid 
  if (!(key in obj)) {}     // ✓ ok 
          
  // Avoid using unnecessary computed property keys on objects.

  const user = { name: 'John Doe' }       // ✓ ok 
          
  // Renaming import, export, and destructured assignments to the

  import { config as config } from './config'     // ✗ avoid 
  import { config } from './config'               // ✓ ok

Testing with confidence


"Before fixing a bug exposed by a high level test, you should replicate the bug with a unit test. Then the unit test ensures the bug stays dead." - Martin Fowler

Snapshot testing with Jest


"After migrating to Jest, our tests started running 6x faster. Running a single test went from about 5 seconds to almost instant." - Rogelio Guzman, Docker

  • Snapshots create a serialized file for your test which can easily be compared using a diff algorithm.
  • Integrates with Babel
  • UI components often change in small and trivial ways. Snapshots are easy to write and effortless to update.
  • In many scenarios, snapshots can remove the need for a unit/integration test, however should not be seen as a complete replacement.

Shadow DOM

"Encapsulated styles are better for you as a developer, and better for you as a platform."

Monica Dinculescu recently summarized common CSS encapsulation strategies:

  • Better naming strategies, i.e. BEM.
  • IFRAMES - you're joking, right?
  • CSS Modules - a smarter way of automating BEM.
  • CSS-in-JS: variations of directly manipulating style attributes, inline css, and compiled stylesheets.
  • Shadow DOM: true CSS encapsulation.

Shadow DOM

Shadow DOM gives you functional boundaries between DOM subtrees. jsfiddle

  • Shadow roots create an encapsulated style boundary.
  • Slots are the mechanism to superimpose (transclude) custom content.
  class MyElement extends HTMLElement {
    constructor() {
      super();
      /* Create a new Shadow Root for your element */
      this._shadowRoot = this.attachShadow({ mode: "open" });
    }
    connectedCallback() {
      /* append your element's template */
      this._shadowRoot.appendChild(document.importNode(elementTemplate.content, true));
    }
  }

CSS Properties and Cascade Rules

CSS Properties cascade similarly to CSS Rules:

:root { --color: blue; } /* default color */
div { --color: green; }
.myclass { --color: red; }
* { color: var(--color); } /* applies the --color property at all the above levels */
        
<p>I inherited blue from the root element!</p>
<div>I got green set directly on me!</div>
<div class="myclass">
  While I got red set directly on me!
  <p>I'm red too, because of inheritance!</p>
</div>

Framework Communication

We communicate with the web component from the framework in three primary ways:

Events 

  this.modal.addEventListener('modal.show', this.monitorModalEvents, false);

Attributes 

<ui-tab title="Tab One" active></ui-tab>

DOM 

  <ui-accordion-template id="collapseOne" aria-labelledby="headingOne">
    <span slot="content" [class.bold]="bold" [innerHTML]="slot"></span>
  </ui-accordion-template>

Framework Extensions (shims)

Sometimes it is desirable to consume components via framework shims to customize these communications. jsfiddle

Shims can help with the following:

  • Making the component feel like a framework component to the consumer
  • Two way binding
  • Dispatching custom events
  • Handling framework mutations
  • Imposing type restrictions
  • Complex components
